# Development Log - 2026-04-04

## Changes

- **[perf]**: Row-level event computation sharing in month view.
  - Lifted `useProcessedWeekEvents` to `HorizontalGridRow`, compute events once per row.
  - Returns `positionedEvents` (passed to `HorizontalGridEventsLayer`) and `dayEventsMap` (passed to `GridCell` via `precomputedEvents` prop).
  - Reduces 48 filter passes over all events to 6 per navigation (one per week row).
  - `HorizontalGridEventsLayer` no longer calls the hook internally — accepts `positionedEvents` as prop.
  - `GridCell` accepts optional `precomputedEvents` prop, skips per-cell `getEventsForDateRange` when provided.
  - Extracted `GroupedColumn` as a memo'd subcomponent for resource week horizontal view (needs its own hook call due to React rules of hooks).
  - Exported `PositionedEvent` type from `position-week-events.ts`.

- **[perf]**: Removed `isDeepEqual` from events sync effect — was running a recursive deep comparison on every events prop change. Replaced with simple reference check.

- **[perf]**: Guarded locale/timezone effects with `useRef` to skip redundant updates on mount and when values haven't changed.

- **[cleanup]**: Deleted `src/lib/utils/deep-equal.ts` and its test file.

## Notes

- The `dayEventsMap` groups events by day key (`YYYY-MM-DD`) using the already-filtered week-level events. This is one pass over a small pre-filtered list, replacing 7 separate `getEventsForDateRange` calls per row.
- `GroupedColumn` exists because React hooks can't be called inside map callbacks — each grouped column (resource week horizontal) needs its own `useProcessedWeekEvents` call with a different `days` array.
- The timezone reactive update (`setCurrentDate` + `setCurrentEvents` with `.tz()`) was kept because events come from user props and don't get re-created when timezone changes. Without it, stored events would display in the original timezone forever.
