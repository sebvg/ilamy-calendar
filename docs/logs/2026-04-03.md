# Development Log - 2026-04-03

## Changes

- **[resource-calendar]**: Added `data` field to `Resource` interface.
  - This allows users to pass custom metadata (e.g., avatar URLs, roles) per resource.
  - The metadata is accessible in custom resource renderers (`renderResource`).
  - Matches the `data` field pattern used in `CalendarEvent`.
- **[documentation]**: Updated `docs/types-and-interfaces.md` to include the new `data` field in the Resource table.
- **[calendar-engine]**: Added visible date range to `onDateChange` event.
  - Signature updated to `(date: Dayjs, range: { start: Dayjs; end: Dayjs }) => void`.
  - Enables efficient backend fetching of events for the currently visible period.
