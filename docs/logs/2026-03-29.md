- **[fix]**: Fixed timezone bug where events appeared in wrong day cells and day labels were shifted when switching timezones. Two root causes:
  1. dayjs `.startOf('week')` drops timezone info when the system's local timezone has a DST transition near the target date. Fixed with a dayjs plugin (`fixTimezoneOffset`) in `dayjs-config.ts` that patches `startOf`/`endOf` to restore the correct timezone offset.
  2. `Intl.DateTimeFormat` with `.toDate()` formatted dates in the system timezone instead of the configured timezone, showing wrong day numbers/names. Replaced all `Intl.DateTimeFormat` usages with dayjs `.format()` which respects the configured timezone.

## Notes

- The TS errors were introduced by `7f9d743 chore: update typescript` which made the compiler stricter.
- All fixes use proper guards/optional chaining instead of non-null assertions, per project code style rules.
- The timezone bug is a known dayjs-timezone issue: `.startOf('week')` internally uses JavaScript's `Date` which picks up the system's local DST transitions, producing wrong UTC offsets for the target timezone. The fix re-applies the timezone with `keepLocalTime=true` when offset drift is detected.
